## 🎯 优先级路线图

### 优先级 1: 让程序真正工作 (必须)

#### 1.1 集成 Windows 消息循环
**目标**: 让程序能够处理 Windows 消息

**为什么重要**:
- 鼠标钩子需要消息循环才能工作
- 系统托盘图标需要消息循环才能显示
- 这是程序能用的前提

**实现方案**:
```rust
// 在 main.rs 中添加消息循环线程
use std::thread;

thread::spawn(|| {
    // 创建隐藏窗口
    let hwnd = create_hidden_window();
    
    // 运行消息循环
    loop {
        let msg = get_message();
        translate_message(&msg);
        dispatch_message(&msg);
    }
});
```

**预计时间**: 2-3 小时

#### 1.2 安装并处理鼠标钩子
**目标**: 实际捕获鼠标移动

**实现步骤**:
1. 在消息循环中调用 `MouseHook::install()`
2. 在钩子回调中发送事件到 PathTracker
3. 连接 PathTracker → GestureRecognizer → IntentFinder → CommandExecutor

**预计时间**: 3-4 小时

#### 1.3 测试端到端流程
**目标**: 验证手势可以触发动作

**测试用例**:
1. 启动程序
2. 执行 "Right → Down" 手势
3. 验证工作站锁定 (Win+L)

**预计时间**: 1-2 小时

**小计**: 6-9 小时 (让程序真正可用)

## 🔧 技术难点提示

### Windows 消息循环集成

**难点**: Rust 的异步运行时 (tokio) 与 Windows 消息循环不兼容

**解决方案**:
1. 在单独的线程中运行消息循环
2. 使用 channel (mpsc) 在异步和同步代码间通信
3. 或者使用 windows-rs 的消息循环包装器

**示例**:
```rust
use std::sync::mpsc;

let (tx, rx) = mpsc::channel();

// 异步主线程
tokio::spawn(async {
    // 等待消息循环线程的信号
    rx.recv().await;
});

// 消息循环线程
thread::spawn(move || {
    let hwnd = create_hidden_window();
    tx.send("ready").unwrap();
    
    loop {
        // 处理消息
    }
});
```

### 鼠标钩子集成

**难点**: 钩子回调必须在同一个线程中安装

**解决方案**:
- 在消息循环线程中安装钩子
- 使用 Arc<Mutex<>> 共享追踪器状态
