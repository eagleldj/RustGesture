# RustGesture - MVP 实施策略

## 🎯 MVP 目标

**最小可用产品 (MVP)**: 用户可以按住中键画个手势,看到动作被执行!

## 📊 当前状态分析

### ✅ 已完成 (90%)
- 完整的手势识别引擎
- 完整的输入模拟系统
- 完整的配置管理
- 完整的意图匹配
- 程序可以启动并加载配置

### ❌ 缺失的关键环节 (10%,但影响 100%)

**问题**: Windows 消息循环集成

**影响**: 
- 无法安装鼠标钩子
- 无法捕获鼠标移动
- 无法执行手势识别
- 程序无法实际工作

**为什么需要消息循环**:
Windows API 要求钩子回调必须在有消息循环的线程中安装,否则钩子无法正常工作。

## 🔧 技术方案

### 方案概述

```
┌─────────────────────────────────────────────────┐
│  main.rs (tokio async)                          │
│                                                  │
│  1. 创建 GestureApp                              │
│  2. 创建 MessageLoop                             │
│  3. 启动消息循环线程  ←──────────────┐          │
│  4. 监听 hook_rx 接收钩子事件        │          │
│  5. 连接 事件 → 追踪器 → 识别器      │          │
└───────────────────────────────────────┼──────────┘
                                            │
┌───────────────────────────────────────▼──────────┐
│  消息循环线程 (sync)                             │
│                                                  │
│  1. 创建隐藏窗口                              │
│  2. 运行消息循环  │        │
│  3. 处理 Windows 消息                          │
│  4. 安装鼠标钩子                              │
│  5. 钩子回调发送事件 → hook_tx                │
└─────────────────────────────────────────────────┘
```

### 核心代码结构

```rust
// main.rs
#[tokio::main]
async fn main() -> Result<()> {
    // ... 初始化代码 ...
    
    // 1. 创建消息循环
    let (mut msg_loop, hook_rx) = winapi::message_loop::MessageLoop::new()?;
    
    // 2. 启动消息循环线程
    let msg_thread = thread::spawn(move || {
        msg_loop.run().unwrap();
    });
    
    // 3. 监听钩子事件
    while let Ok(msg) = hook_rx.recv() {
        match msg {
            HookMessage::MouseMoved { x, y } => {
                // 发送到追踪器
            }
            // ... 处理其他事件 ...
        }
    }
}
```

## 📌 关键注意事项

1. **线程安全**: 
   - 消息循环在单独线程
   - 使用 Arc<Mutex<>> 共享状态
   - 使用 channel (mpsc) 线程间通信

2. **生命周期**:
   - 消息循环线程必须在 tokio 运行时之前启动
   - 确保线程在退出前正确清理

3. **错误处理**:
   - 消息循环崩溃不影响主程序
   - 钩子失败时优雅降级
